Parser Help

Input

While the SABNF grammar can easily define sentences and phrases with arbitrary binary characters, with this web interface input can only be entered through an input <textarea>; reading files is not possible. The Parser -> Input tab is a <textarea> for the parser’s input. Input to the parser is typed or pasted into this <textarea>.

Because of the <textarea> and the variations of how they are handled across the different browsers, there is no reliable way to handle non-printing ASCII characters. However, a configuration option is available to allow non-printing characters to be entered with an escaped, hexadecimal format. Whether in ASCII-only mode (default) or escaped character mode, only character codes TAB(9), LF(10), CR(13) and 32‐126 are allowed. Any other character will be rejected as an input error.

In escaped character mode, the escape character is the grave accent (`). This choice was made because the grave accent is rarely used in English, ASCII text and does not require the shift key. The backslash (\) was rejected because of annoying conflicts with the JavaScript string escape character, the percent (%) was rejected because of conflicts with the ABNF grammar syntax, and the tilde (~) was rejected because it requires the shift key. The escaped encoding rules are:

  • `` – literal grave accent (`)
  • `xhh – 8-bit integer
  • `Xhhhh – 16-bit integer
  • `uhhhhhhhh – 32-bit integer
  • h – hexadecimal digit, 0-9, a-f, or A-F

Grave accent (`) used in any context other than `x, `X, `u, or `` is an input error. Double clicking within the input <textarea> will check the input for errors and provide an annotated display in the Phrases tab.
